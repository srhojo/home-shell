# Documentación del proyecto

Esta carpeta contiene la documentación fundacional del proyecto.
Define los conceptos, reglas y decisiones arquitectónicas que rigen el sistema.

El objetivo es que el proyecto pueda retomarse en cualquier momento
sin necesidad de reanalizar decisiones pasadas.

---

## Documentos

- **ARCHITECTURE.md**
  - Arquitectura general
  - Principios de diseño
  - Proceso de arranque

- **MODULES.md**
  - Qué es un módulo
  - Qué no es un módulo
  - Contrato (manifest)
  - Independencia y estados

---

## Principio clave

> Ninguna decisión estructural debería tomarse fuera de estos documentos.

Si algo no encaja aquí, probablemente no deba implementarse.
